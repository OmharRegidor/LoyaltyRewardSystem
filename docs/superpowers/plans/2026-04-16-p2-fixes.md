## Task 1: Account Deletion — Clean Up Stamp & POS Data

**Files:**
- Create: `supabase/migrations/20260416100000_cascade_delete_stamp_pos_data.sql`
- Modify: `apps/web/app/api/account/delete/route.ts`

- [ ] **Step 1: Create migration to add explicit deletes for stamp/POS data**

The safest approach is to add explicit DELETE statements in the account deletion route rather than altering FK constraints (which could have side effects on other delete paths). But we should also ensure FK cascades are set up properly for future safety.

Create migration:

```sql
-- 20260416100000_cascade_delete_stamp_pos_data.sql
-- Ensure stamp_entries, stamp_cards, and pos-related data are cleaned up
-- when a customer is deleted.

-- stamp_entries references stamp_cards (already cascades via stamp_card_id FK)
-- stamp_cards references customers via customer_id — add CASCADE if missing
DO $$
BEGIN
  -- Drop and recreate stamp_cards customer_id FK with CASCADE
  IF EXISTS (
    SELECT 1 FROM information_schema.table_constraints
    WHERE constraint_name = 'stamp_cards_customer_id_fkey'
    AND table_name = 'stamp_cards'
  ) THEN
    ALTER TABLE stamp_cards DROP CONSTRAINT stamp_cards_customer_id_fkey;
    ALTER TABLE stamp_cards ADD CONSTRAINT stamp_cards_customer_id_fkey
      FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE CASCADE;
  END IF;

  -- Drop and recreate pos_sales customer_id FK with CASCADE
  IF EXISTS (
    SELECT 1 FROM information_schema.table_constraints
    WHERE constraint_name = 'pos_sales_customer_id_fkey'
    AND table_name = 'pos_sales'
  ) THEN
    ALTER TABLE pos_sales DROP CONSTRAINT pos_sales_customer_id_fkey;
    ALTER TABLE pos_sales ADD CONSTRAINT pos_sales_customer_id_fkey
      FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE SET NULL;
  END IF;
END;
$$;
```

Note: `pos_sales` uses SET NULL instead of CASCADE because sales records should be preserved for business accounting — just de-link from the deleted customer.

- [ ] **Step 2: Update account deletion route to explicitly delete stamp data**

Read `apps/web/app/api/account/delete/route.ts`. Add stamp data cleanup before the existing `customer_businesses` deletion. The current deletion order is: customer_businesses → transactions → customers.

Add before the customer_businesses delete:


Also parallelize the independent delete operations using `Promise.all()`:

```ts

      await Promise.all([
        adminClient.from('stamp_cards').delete().eq('customer_id', customer.id),
        adminClient.from('customer_businesses').delete().eq('customer_id', customer.id),
        adminClient.from('transactions').delete().eq('customer_id', customer.id),
      ]);

      // Delete customer last (after all FK references removed)
      await adminClient.from('customers').delete().eq('id', customer.id);
    }
```
